# Patch — CSS split, per-component subpaths, publish-readiness

Status: proposed
Owner: @kokjinsam
Branch target: `main`
Target version: `0.1.0` (breaking minor bump from `0.0.1`)

## Goal

Three intertwined changes that, together, make `@vrmn/komorebi` ready to publish to npm with a clean public surface:

1. **Split CSS** — replace `src/styles/index.css` with three discrete files (`globals.css`, `default-theme.css`, `komorebi.css`) plus a Tailwind preset (`tailwind.css`). Each has a single, named responsibility.
2. **Per-component subpath imports** — `import { Button } from "@vrmn/komorebi/Button"` works alongside the existing barrel `import { Button } from "@vrmn/komorebi"`.
3. **CSS pipeline change** — CSS files are shipped raw (no Tailwind compilation at library build time). Consumer's Tailwind v4 setup compiles utilities by scanning the published JS via `@source`.

This is a hard-cut refactor — no back-compat shims for the old `dist/komorebi.css` or the old `src/styles/index.css` import path. Version bump from `0.0.1` → `0.1.0` carries the signal.

## Decisions (locked in via interview)

| Topic | Decision |
| --- | --- |
| CSS split | **Strict**: `default-theme.css` = tokens only (`:root` + `.dark`). `globals.css` = base + `@theme inline` bridge + body + media query. `komorebi.css` = handwritten component CSS only. |
| Component CSS source-of-truth | `komorebi.css` aggregates via `@import`. Toast/index.tsx no longer imports its CSS in TS. |
| Tailwind preset | Separate `tailwind.css` containing `@import "tailwindcss"` + `@plugin` lines + `@variant dark`. **No `@source`** — consumer adds their own. |
| Tailwind utilities | NOT precompiled into the library. Consumer's Tailwind v4 generates them by scanning `node_modules/@vrmn/komorebi` via consumer-supplied `@source`. |
| CSS build | None. CSS files are copied verbatim from `src/styles/` to `dist/styles/`. Vite's auto CSS bundling is disabled. |
| TW plugin deps | `tailwindcss`, `tailwindcss-react-aria-components`, `tailwindcss-animate` become `peerDependencies` ONLY. Removed from `devDependencies`. |
| Subpath shape | Flat: `@vrmn/komorebi/Button`, `@vrmn/komorebi/Toast`, …, `@vrmn/komorebi/utils`. |
| Barrel | Kept. Both `@vrmn/komorebi` and `@vrmn/komorebi/<Name>` work. |
| Build entries | Per-component Vite entries with folder output: `dist/Button/index.js`, `dist/Toast/index.js`, … plus `dist/index.js` (barrel) and `dist/utils/index.js`. Shared chunks in `dist/chunks/`. |
| `exports` map | Enumerated per entry (~58 entries). Generated by a `scripts/sync-exports.ts` script that reads `src/components/`. CI verifies the file matches what the script would emit. |
| Vite entries | Generated dynamically at config-evaluation time from `readdirSync("src/components")`. |
| `"use client"` banner | Keep on every entry AND every chunk (current `output.banner` behavior). |
| Dark mode trigger | `.dark` class only. No `prefers-color-scheme` fallback. |
| Token names | **Public API contract** from 0.1.0 onward. Renames require a major bump. |
| Storybook preview | Imports all four files: `tailwind.css`, `globals.css`, `default-theme.css`, `komorebi.css`. |
| README | Out of scope for this patch. Tracked separately. |

## CSS file contracts

### `default-theme.css` — tokens only

Contains ONLY CSS custom property declarations. No selectors other than `:root` and `.dark`. No Tailwind directives. No body styles. No media queries.

```css
:root {
  /* Surfaces */
  --background: oklch(1 0 0);
  --foreground: oklch(0.145 0 0);
  --card: oklch(1 0 0);
  --card-foreground: oklch(0.145 0 0);
  --popover: oklch(1 0 0);
  --popover-foreground: oklch(0.145 0 0);

  /* Roles */
  --primary: oklch(0.205 0 0);
  --primary-foreground: oklch(0.985 0 0);
  --secondary: oklch(0.97 0 0);
  --secondary-foreground: oklch(0.205 0 0);
  --muted: oklch(0.97 0 0);
  --muted-foreground: oklch(0.556 0 0);
  --accent: oklch(0.97 0 0);
  --accent-foreground: oklch(0.205 0 0);
  --destructive: oklch(0.577 0.245 27.325);
  --destructive-foreground: oklch(0.985 0 0);

  /* Structural */
  --border: oklch(0.922 0 0);
  --input: oklch(0.922 0 0);
  --ring: oklch(0.708 0 0);
  --ring-offset: var(--background);
  --radius: 0.625rem;

  /* Data viz */
  --chart-1: oklch(0.87 0 0);
  /* …chart-2…5 */

  /* Sidebar */
  --sidebar: oklch(0.985 0 0);
  /* …sidebar-* */
}

.dark {
  --background: oklch(0.145 0 0);
  /* … all dark overrides … */
}
```

A consumer who wants only the token contract (no body styles, no @theme bridge) imports just this file.

### `globals.css` — base + @theme bridge

Contains:
- The `@theme inline` block (the bridge mapping `--background` → `--color-background` etc.)
- Font-slot `@theme inline` entries (`--font-sans`, `--font-heading`, `--font-mono`, `--font-serif`).
- The radius scale (`--radius-sm` … `--radius-4xl`).
- Body styles (`body { background: var(--background); color: var(--foreground); color-scheme: dark light; }`).
- The high-resolution media query (`@media (min-resolution: 200dpi)`).

Does NOT contain:
- Tailwind `@import` or `@plugin` directives — those live in `tailwind.css`.
- Token values — those live in `default-theme.css`.
- Component-specific CSS — those live in `komorebi.css`.

The bridge mappings are identical for any theme; a custom theme just supplies different `--background` values, never redeclares the bridge. That is why the bridge belongs here, not in `default-theme.css`.

### `komorebi.css` — handwritten component CSS

Aggregates all component-scoped CSS via relative `@import`:

```css
/* src/styles/komorebi.css */
@import "./Toast.css";
/* future component CSS files added here */
```

Currently only `Toast.css` (view-transition animations) is component-scoped. The file exists as the documented import path consumers wire up — future components add their CSS here without touching consumer integration.

### `tailwind.css` — Tailwind v4 preset

```css
@import "tailwindcss";
@plugin "tailwindcss-react-aria-components";
@plugin "tailwindcss-animate";
@variant dark (&:where(.dark, .dark *));
```

No `@source` — consumer must declare their own `@source "../node_modules/@vrmn/komorebi"` (or wherever komorebi resolves in their tree). Reason: `@source` paths are relative to the file that contains them, and we cannot predict the consumer's CSS location.

This file is *optional* for consumers who already have Tailwind + the two plugins wired into their own app CSS. We ship it for the common case where they don't.

## Consumer integration

Expected consumer CSS (minimal):

```css
/* consumer's app.css */
@import "@vrmn/komorebi/styles/tailwind.css";
@source "../node_modules/@vrmn/komorebi";

@import "@vrmn/komorebi/styles/globals.css";
@import "@vrmn/komorebi/styles/default-theme.css";
@import "@vrmn/komorebi/styles/komorebi.css";
```

Variants:
- **Consumer with their own theme**: skip `default-theme.css`, provide their own `:root`/`.dark` with the same variable names (token contract — see below).
- **Consumer with their own Tailwind setup**: skip `tailwind.css`, but still must declare a `@source` for komorebi and ensure the two TW plugins are loaded.
- **Consumer who wants tokens only (e.g. for non-React app)**: import just `default-theme.css`.

Component imports work in two equivalent ways:

```ts
// barrel — preferred for most consumers
import { Button, Toast, IconProvider } from "@vrmn/komorebi"

// per-component subpath — preferred for max tree-shake safety
import { Button } from "@vrmn/komorebi/Button"
import { Toast } from "@vrmn/komorebi/Toast"
import { cn, IconProvider } from "@vrmn/komorebi/utils"
```

### 4. Add per-component entries to `vite.config.ts`

Generate entries dynamically from the filesystem:

```ts
import { readdirSync, statSync } from "node:fs"
import { resolve, dirname } from "node:path"
import { fileURLToPath } from "node:url"
import { defineConfig } from "vite"
import dts from "vite-plugin-dts"
import pkg from "./package.json"

const __dirname = dirname(fileURLToPath(import.meta.url))

const componentsDir = resolve(__dirname, "src/components")
const componentEntries = Object.fromEntries(
  readdirSync(componentsDir)
    .filter(name => statSync(resolve(componentsDir, name)).isDirectory())
    .map(name => [name, resolve(componentsDir, name, "index.tsx")])
)

export default defineConfig(({ mode }) => {
  const isDev = mode === "development"

  return {
    build: {
      lib: {
        entry: {
          index: resolve(__dirname, "src/index.ts"),
          "utils/index": resolve(__dirname, "src/utils/index.ts"),
          ...componentEntries
        },
        // Preserve folder structure: <name>/index.js
        fileName: (_format, entryName) =>
          entryName === "index"
            ? "index.js"
            : entryName.includes("/")
              ? `${entryName}.js`
              : `${entryName}/index.js`,
        formats: ["es"]
      },
      cssCodeSplit: false,
      // Critical: disable Vite's CSS asset emission. Our CSS is copied
      // verbatim via the plugin below, not bundled.
      rollupOptions: {
        external: [
          ...Object.keys(pkg.peerDependencies ?? {}),
          "react/jsx-runtime"
        ],
        output: {
          preserveModules: false,
          chunkFileNames: (chunkInfo) => {
            const name = chunkInfo.name.replace(/[-_]+$/, "") || "chunk"
            return `chunks/${name}-[hash:16].js`
          },
          hashCharacters: "base36",
          hoistTransitiveImports: false,
          banner: '"use client";\n',
          globals: {
            "react": "React",
            "react-dom": "ReactDOM",
            "react/jsx-runtime": "jsxRuntime"
          },
          // Each entry into its own folder; chunks shared in chunks/
          entryFileNames: (chunkInfo) => {
            if (chunkInfo.name === "index") return "index.js"
            if (chunkInfo.name === "utils/index") return "utils/index.js"
            return `${chunkInfo.name}/index.js`
          }
        },
        cache: isDev
      },
      sourcemap: isDev ? "inline" : true,
      minify: isDev ? false : "esbuild",
      emptyOutDir: !isDev,
      watch: isDev
        ? {
            exclude: ["**/*.test.*", "**/*.stories.*", "**/__tests__/**"],
            include: "src/**"
          }
        : undefined
    },
    plugins: [
      dts({
        compilerOptions: {
          incremental: isDev,
          tsBuildInfoFile: isDev ? "./.tsbuildinfo" : undefined
        },
        exclude: ["**/*.test.ts", "**/*.test.tsx", "**/*.stories.tsx"],
        include: ["src/**/*"],
        // Match the JS output layout: dist/<Name>/index.d.ts
        entryRoot: "src",
        outDir: "dist"
      }),
      copyCssPlugin()    // see step 5
    ],
    resolve: {
      alias: {
        "@": resolve(__dirname, "src")
      }
    }
  }
})
```

Notes:
- `dts` config currently writes to `dist/src/index.d.ts`; change `entryRoot: "src"` so it writes `dist/index.d.ts` and `dist/Button/index.d.ts` matching the JS paths. Update `package.json` `types` field accordingly.
- The double specification of `entry.fileName` and `output.entryFileNames` is intentional belt-and-suspenders — Vite resolves the former when constructing rollup options; the latter is the actual rollup directive.

### 5. Add a small Vite plugin to copy `src/styles/*.css` → `dist/styles/`

`vite.config.ts` (inline or in `scripts/copy-css-plugin.ts`):

```ts
function copyCssPlugin(): Plugin {
  const stylesSrc = resolve(__dirname, "src/styles")
  const stylesOut = resolve(__dirname, "dist/styles")
  return {
    name: "komorebi-copy-css",
    apply: "build",
    closeBundle() {
      mkdirSync(stylesOut, { recursive: true })
      for (const f of readdirSync(stylesSrc)) {
        if (f.endsWith(".css")) {
          copyFileSync(resolve(stylesSrc, f), resolve(stylesOut, f))
        }
      }
    }
  }
}
```

No transformation, no `@import` flattening, no minification. The published `dist/styles/komorebi.css` is byte-identical to `src/styles/komorebi.css` and its relative `@import "./Toast.css"` resolves against the published sibling file.

### 9. Add `scripts/sync-exports.ts`

```ts
#!/usr/bin/env tsx
import { readdirSync, statSync, readFileSync, writeFileSync } from "node:fs"
import { resolve, dirname } from "node:path"
import { fileURLToPath } from "node:url"

const __dirname = dirname(fileURLToPath(import.meta.url))
const root = resolve(__dirname, "..")
const componentsDir = resolve(root, "src/components")
const pkgPath = resolve(root, "package.json")
const check = process.argv.includes("--check")

const components = readdirSync(componentsDir)
  .filter(n => statSync(resolve(componentsDir, n)).isDirectory())
  .sort()

const componentExports = Object.fromEntries(
  components.map(name => [
    `./${name}`,
    {
      types: `./dist/${name}/index.d.ts`,
      import: `./dist/${name}/index.js`
    }
  ])
)

const exportsMap = {
  ".": {
    types: "./dist/index.d.ts",
    import: "./dist/index.js"
  },
  "./utils": {
    types: "./dist/utils/index.d.ts",
    import: "./dist/utils/index.js"
  },
  ...componentExports,
  "./styles/tailwind.css": "./dist/styles/tailwind.css",
  "./styles/globals.css": "./dist/styles/globals.css",
  "./styles/default-theme.css": "./dist/styles/default-theme.css",
  "./styles/komorebi.css": "./dist/styles/komorebi.css",
  "./package.json": "./package.json"
}

const pkg = JSON.parse(readFileSync(pkgPath, "utf8"))
const current = JSON.stringify(pkg.exports, null, 2)
const desired = JSON.stringify(exportsMap, null, 2)

if (check) {
  if (current !== desired) {
    console.error("package.json exports is out of sync with src/components/.")
    console.error("Run `pnpm sync-exports` to fix.")
    process.exit(1)
  }
  return
}

pkg.exports = exportsMap
writeFileSync(pkgPath, JSON.stringify(pkg, null, 2) + "\n")
```

The script is the source of truth for the exports map. `prepublishOnly` invokes `--check` so a forgotten sync blocks publish.

## Token contract (stable as of 0.1.0)

The following CSS variable names are part of the package's public API. Renames or removals require a major version bump.

### Surfaces
`--background`, `--foreground`, `--card`, `--card-foreground`, `--popover`, `--popover-foreground`

### Roles
`--primary`, `--primary-foreground`, `--secondary`, `--secondary-foreground`, `--muted`, `--muted-foreground`, `--accent`, `--accent-foreground`, `--destructive`, `--destructive-foreground`

### Structural
`--border`, `--input`, `--ring`, `--ring-offset`, `--radius`

### Data viz
`--chart-1`, `--chart-2`, `--chart-3`, `--chart-4`, `--chart-5`

### Sidebar
`--sidebar`, `--sidebar-foreground`, `--sidebar-primary`, `--sidebar-primary-foreground`, `--sidebar-accent`, `--sidebar-accent-foreground`, `--sidebar-border`, `--sidebar-ring`

### Derived radii (defined in globals.css `@theme inline`)
`--radius-sm`, `--radius-md`, `--radius-lg`, `--radius-xl`, `--radius-2xl`, `--radius-3xl`, `--radius-4xl`

### Font slots (defined in globals.css `@theme inline`, all default to `inherit` / system stacks)
`--font-sans`, `--font-heading`, `--font-mono`, `--font-serif`

### Tailwind utility bridge variables (defined in globals.css `@theme inline`)

Each `--color-<role>` maps to its corresponding `--<role>` variable. Renaming a token without also renaming its bridge entry is invalid.

### Dark mode

`.dark` class on any ancestor activates the dark token set. No `prefers-color-scheme` fallback is provided. Custom themes that want auto-detection wrap their own `:root { @media (prefers-color-scheme: dark) { … } }` rules.

## Risks and sharp edges

- **Tailwind v4 `@source` resolution.** Consumer's `@source "../node_modules/@vrmn/komorebi"` must resolve from the directory of their app CSS. If pnpm hoists differently (`.pnpm/`), the path may need adjusting. Document the typical patterns (npm flat, pnpm hoisted, pnpm strict) in README follow-up.
- **`tailwindcss-react-aria-components` and `tailwindcss-animate` as peerDeps only.** Storybook builds in this repo will fail without those installed. pnpm's auto peer installation typically covers it, but verify after the devDep removal; if it breaks, the fallback is to keep them as devDeps AND peerDeps.
- **`dts` plugin output paths.** Setting `entryRoot: "src"` is the canonical fix to flatten `dist/src/…` → `dist/…`, but verify that types files are emitted under `dist/Button/index.d.ts` (matching JS) and not `dist/components/Button/index.d.ts`. If the dts plugin doesn't honor `entryRoot` cleanly, fall back to a post-build move script.
- **Per-component bundles size**. 53 entries with shared chunks should produce a manageable `chunks/` folder, but watch for chunk explosion. Run `du -sh dist/` after build and compare to the current single-bundle size; flag if >2× growth.
- **`"use client"` banner on chunk files.** Some bundlers (notably older webpack) get confused by directives in shared chunks. Keeping the banner everywhere is the safer default for Next.js App Router; if any consumer reports issues, revisit.
- **Token contract lock-in.** Once shipped at 0.1.0, renaming `--background` to `--bg` is a major bump. This is fine but worth knowing.
- **No fallback for forgotten `komorebi.css` import.** A consumer who imports `Toast` but forgets `@import "@vrmn/komorebi/styles/komorebi.css"` will see broken animations without an error. The JSDoc on `Toast` and a README note are the only mitigations.
- **`dist/komorebi.css` (root-level) is gone.** Anyone with a stale `@import "@vrmn/komorebi/dist/komorebi.css"` pinned to 0.0.1 will break on upgrade. Acceptable — the version bump signals breaking changes.
- **`sideEffects` change.** Dropping the `"*.css"` glob means bundlers that previously tree-shook component-level CSS imports no longer have to. Since we removed those imports, this is neutral, but verify no other component sneaks in a TS-side CSS import in the future (add a lint rule? out of scope).

## Out of scope

- README "Getting Started" overhaul (follow-up patch).
- Light/dark auto-detection via `prefers-color-scheme`.
- A `@vrmn/komorebi/preset` Tailwind config helper for v3 consumers (we're v4-only).
- Renaming any existing CSS variable, even to align with shadcn/Luma conventions.
- Splitting `komorebi.css` further by category (overlays.css, forms.css). One aggregate file is enough until proven otherwise.
- Bundling `@phosphor-icons/react` or any peer dep into the library.
- Adding tests for the build pipeline.
- Replacing the build with `tsup`, `rolldown`, or other bundler swaps.
